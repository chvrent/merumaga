**開発指示書**										
										
- **目的**: このブックに追加済みの `app_*` シートをデータソースとして、Apps Script のWebアプリで「配信マスタ検索・PR本文確認・休暇対応確認・商品系条件確認」ができるアプリを作成する。										
- **前提データ**: 元シートではなく、アプリ用に整形済みの `app_schedule`, `app_pr`, `app_pr_targets`, `app_holidays`, `app_product`, `app_readme` を読む。										
- **実装方式**: Apps Script Webアプリは `doGet(e)` または `doPost(e)` を持ち、`HtmlOutput` または `TextOutput` を返す必要があるため、画面UIは `HtmlService`、画面からサーバー処理呼び出しは `google.script.run` を使う構成にする。 ([ウェブアプリ  |  Apps Script  |  Google for Developers](https://developers.google.com/apps-script/guides/web?hl=ja&utm_source=openai))										
										
**推奨構成**										
- `Code.gs`: `doGet()`, API入口、共通レスポンス処理。										
- `DataService.gs`: スプレッドシート読み込み、シート→JSON変換、検索/絞り込み。										
- `Index.html`: 画面本体。										
- `Styles.html`: CSS。										
- `Client.html`: クライアント側JavaScript。										
- データ取得は日付表示を保つため `getDisplayValues()` を使用する。Google公式ドキュメントでも、表示値を文字列として取得する用途に `getDisplayValues()` / `getDisplayValue()` が示されている。 ([Class Range  |  Apps Script  |  Google for Developers](https://developers.google.com/apps-script/reference/spreadsheet/range?utm_source=openai))										
										
**利用シート**										
- `app_schedule`: 配信マスタ。主キーは `id`、表示名は `mail_name`、日付は `start_date` / `end_date`。										
- `app_pr`: PR本文マスタ。主キーは `pr_id`、本文は `pr_text`。										
- `app_pr_targets`: PRと対象メルマガの1対多対応表。`pr_id` と `mail_name` で紐付け。										
- `app_holidays`: 長期休暇・祝日対応。`delivery_date`, `mail_name`, `delivery_status` を使用。										
- `app_product`: 商品系メルマガの条件・通数マスタ。`mail_name`, `after_setting`, `after_volume` を使用。										
										
**サーバー側API仕様**										
```javascript										
function doGet(e) {										
  return HtmlService.createTemplateFromFile('Index')										
    .evaluate()										
    .setTitle('メルマガ配信マスタ')										
    .addMetaTag('viewport', 'width=device-width, initial-scale=1');										
}										
										
function include(filename) {										
  return HtmlService.createHtmlOutputFromFile(filename).getContent();										
}										
										
function getInitialData() {										
  return {										
    schedule: getSheetObjects_('app_schedule'),										
    pr: getSheetObjects_('app_pr'),										
    prTargets: getSheetObjects_('app_pr_targets'),										
    holidays: getSheetObjects_('app_holidays'),										
    product: getSheetObjects_('app_product')										
  };										
}										
										
function getSheetObjects_(sheetName) {										
  const ss = SpreadsheetApp.getActiveSpreadsheet();										
  const sheet = ss.getSheetByName(sheetName);										
  if (!sheet) throw new Error(`Sheet not found: ${sheetName}`);										
										
  const values = sheet.getDataRange().getDisplayValues();										
  const headers = values.shift();										
										
  return values										
    .filter(row => row.some(v => v !== ''))										
    .map(row => {										
      const obj = {};										
      headers.forEach((h, i) => obj[h] = row[i] || '');										
      return obj;										
    });										
}										
```										
										
**画面機能要件**										
- **配信マスタ検索**: `mail_name`, `weekday`, `hour`, `category`, `format`, `assignee`, `reviewer` で検索/フィルタ。										
- **有効/停止フィルタ**: `is_inactive` が `TRUE` / `true` の行は「停止・非表示」扱いにできるようトグルを用意。										
- **PR確認**: `app_pr_targets.mail_name` から該当 `pr_id` を探し、`app_pr.pr_text` と `insert_position` を表示。										
- **休暇対応確認**: `mail_name` または `delivery_date` で `app_holidays` を検索し、`delivery_status` と `contact_note` を表示。										
- **商品条件確認**: `mail_name` で `app_product` を検索し、`after_setting`, `after_volume`, `notes` を表示。										
- **コピー機能**: PR本文、配信条件、対象メルマガ名をワンクリックでコピーできるボタンを設置。										
										
**クライアント側呼び出し例**										
```html										
<script>										
let APP_DATA = {};										
										
function loadData() {										
  showLoading(true);										
										
  google.script.run										
    .withSuccessHandler(data => {										
      APP_DATA = data;										
      renderSchedule(data.schedule);										
      showLoading(false);										
    })										
    .withFailureHandler(error => {										
      showLoading(false);										
      alert('データ取得に失敗しました: ' + error.message);										
    })										
    .getInitialData();										
}										
										
function isActiveRow(row) {										
  return String(row.is_inactive || '').toLowerCase() !== 'true';										
}										
										
function searchSchedule(keyword) {										
  const q = String(keyword || '').toLowerCase();										
  const rows = APP_DATA.schedule || [];										
										
  return rows.filter(row => {										
    if (!isActiveRow(row)) return false;										
    return [										
      row.mail_name,										
      row.weekday,										
      row.hour,										
      row.category,										
      row.format,										
      row.assignee,										
      row.reviewer										
    ].join(' ').toLowerCase().includes(q);										
  });										
}										
</script>										
```

**デプロイ手順**										
- スプレッドシートから **拡張機能 → Apps Script** を開く。										
- `Code.gs`, `DataService.gs`, `Index.html`, `Styles.html`, `Client.html` を作成。										
#ERROR!										
- **デプロイ → 新しいデプロイ → 種類: ウェブアプリ** を選択。										
#ERROR!										
- `/exec` URL を共有し、更新時は新バージョンで再デプロイ。										
										
**注意点**										
- 元シートではなく、必ず `app_*` シートを読む。										
- 日付は表示値 `yyyy-mm-dd` をそのまま使う。										
- `#REF!` は出力シート側で空欄化済みだが、空欄チェックは必ず入れる。										
- PRは `app_pr` と `app_pr_targets` を分けて扱い、1つのPRが複数メルマガに紐づく前提で実装する。
