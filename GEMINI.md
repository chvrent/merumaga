# メルマガスケジュール管理プロジェクト

## 1. 概要
メルマガの配信スケジュールをGoogleスプレッドシート上で効率的に作成・管理するシステム。
「関数による動的なスケジュール抽出」と「GASによる納品用シートの生成」を組み合わせたハイブリッド運用を行う。

## 2. 運用フロー
1.  **マスタ更新**: 「リスト」「特殊配信マスタ」「新規」の各シートに最新の配信ルールを記載する。
    - PRは `PR管理` に登録し、`📅 スケジュール管理 → PR管理 → リスト(PR列)へ反映` で対象メルマガへ紐付ける。
    - `PR管理` に書いた「PRが入るメルマガ」は完全一致。`PR管理` の `完全一致チェック(リスト未登録)` が空でない場合、`リスト` に存在しないメルマガ名が混ざっている。
    - `PR管理` のメルマガ名列（G列以降）が増えても、`リスト!PR` が壊れないように内部ヘルパー列 `PR_TARGET_MAIL(内部)` / `PR_TARGET_PR(内部)` を使って参照する（列位置ではなくヘッダー名で追従）。
2.  **編集版の作成**: `yy/m/d_編集版` シートにて、スプレッドシート関数（FILTER等）を用いてマスタから予定を抽出する。
    - 配信停止でも抽出は残し、配信停止の行はグレー表示にする。
    - 色分けは、各マスタ側の背景色を編集版へ同期する（メニュー実行）。
3.  **確定版の生成**:
     *   メニューの「📅 スケジュール管理」→「確定版を作成（空行のみ削除）」を実行。
     *   全曜日が空の行を自動削除し、リンクやデザイン（太字・色）を保持した `yy/m/d_確定版` が生成される。

## 3. シート定義とシステム役割（仕様書）

本システムは、マスタから確定版へデータを流し込む以下の構造で定義される。

| シート分類 | 対象シート名 | 役割 | 編集ルール |
| :--- | :--- | :--- | :--- |
| **マスタ** | リスト, 新規, 特殊配信マスタ, 設定者マスタ | 配信ルールの正本 | **手動入力が必要**。ここを更新すると全編集版に反映される。 |
| **編集版** | yy/m/d_編集版 | スケジュールの可視化 | **抽出専用**。数式によりマスタから自動転記される（手入力禁止）。 |
| **確定版** | yy/m/d_確定版 | 納品用・記録用 | 静的な値として固定。修正は編集版に戻って行う。 |

### 詳細仕様
- **マスタ**: 抽出の根拠。表記ゆれ防止のため「入力規則」が適用されている。
- **編集版**: ロジック更新により、マスタの内容と同期する。数式が全枠に動的に配置され、配信件数に応じて拡張される。
- **確定版**: 編集版の計算結果を「値」として確定させ、不要な空行を削除した「配信時点のバックアップ」。

### 次回修正の当たりどころ（チェックリスト）
- 編集版の抽出: 毎週=リスト、新規=新規、特殊=特殊配信マスタ。配信停止でも抽出は残し、見た目だけグレー。
- 列追加への耐性: 固定列参照は避け、ヘッダー名/MATCH や「チェック列の手前」などで動的に範囲を決める。
- PR: `PR管理` のメルマガ名は完全一致。未登録は `完全一致チェック(リスト未登録)` で検知。曖昧（同名が複数PR等）はエラーで止める。

## 6. 開発・運用ルールと失敗を防ぐ対策（2026/04/30改訂）

本プロジェクトにおける技術的判断の失敗を防ぐため、以下のルールを厳守する。

### 完了定義 (Definition of Done) - ※更新の必須条件
修正やロジック更新の「完了」は、以下の全項目を満たした状態を指す。
1. **ルール整合性**: マスタの配信対象データが、編集版シートの該当枠に過不足なく抽出されていること。
2. **完全抽出**: マスタのデータ数と、表示されている枠数（スロット）が合致していること。
3. **データの一貫性**: 設定者・確認者が「設定者マスタ」に従い、正確に表示されていること。
4. **エラー皆無**: `#VALUE!`, `#N/A`, `#REF!` 等のエラーが全セルで発生していないこと。

### 失敗を防ぐための基本ルール
1. **検証の義務化 (検証マスト)**:
    - 上記の完了定義を満たすことを、検証スクリプトまたは目視確認で証明してからプッシュする。
    - 抽出結果が0件やエラーになる場合は「未完了」とする。
2. **自動化プロセス**:
    - コード変更時は必ず検証用スクリプトを実行し、結果（抽出件数・データの一致）をユーザーに提示し、合意を得てから `clasp push` を行う。
3. **MDの更新 (更新マスト)**:
    - 仕組みや仕様に変更があった場合は、作業終了時に必ず `GEMINI.md` を更新する。
4. **バックアップの徹底**:
    - 作業開始前にバックアップを作成し、いつでも復元可能な状態を維持する。
3.  **バックアップの徹底**:
    - 作業開始前にバックアップを作成し、いつでも復元可能な状態を維持する。
    - 誤った修正を適用した場合は、即座にバックアップから復旧し、原因をドキュメントに記録する。

### トラブル発生時の対応ルール
- **「全部エラー」「データが出ない」等の事象が発生した場合**:
    - 即座に古い状態へ戻し、無理な修正を続けない。
    - 成功していた時点のファイルとの差分を再確認し、論理的な整合性を再構築する。
- **指示とフィードバック**:
    - 運用者の指示（例：「戻した」「ちゃんと検証して」）を最優先とし、不具合を解消するまで次のタスクへ進まない。


## 特殊配信マスタ（整理方針）
- 推奨は「縦持ち（1行=1配信）」。
  - 同じ(曜日,時間,サイクル)で2本以上ある場合は複数行にする（メイン/2/関西で列を増やさない）。
  - 必須ヘッダー: `曜日` / `時間` / `サイクル` / `メルマガ内容`
  - 編集版へ転記: `通数` / `設定者` / `確認者`
  - 停止は抽出から除外しない（`配信停止=TRUE` は見た目だけグレー）。
- 旧形式（`メルマガ内容(メイン)` / `メルマガ内容(2)` / `関西(3のみ)`）も当面は互換で動くが、新形式へ寄せる。

### 配信ルール（マスタ）
- **隔週配信**: A週（緑）とB週（ピンク）が毎週交互に配信される。
- **ポジションマッチ**: 
  - A1→A2→A3、B1→B2→B3 の3週1サイクル。
  - サイクルが「3」の時のみ、「◇ポジションマッチ 関西」を追加。
- **特定日配信**: 第2・第4日曜の「IT派遣」など、日付連動で抽出。
- **配信停止**: 編集版では抽出対象から外さず、見た目だけグレーアウトして運用する。

## 4. レイアウトとマスタ連携仕様

### 編集版シートのグリッド構造
- **横軸**: C列(火)から始まり、5列単位で7日分（火〜月）が並ぶ。
- **縦軸**: A列(時間)、B列(セクション名)が制御軸。
- **データブロック**:
  - 各日(5列): `メルマガ内容(C/H/...)` / `通数` / `計` / `設定者(F/K/...)` / `確認者(G/L/...)`

### マスタ・データ連携マッピング表
抽出ロジック（`applyLatestSystemLogic`）は以下のマッピングでデータを参照する。

| セクション | 抽出対象 | メルマガ内容 | 設定者 (Setter) | 確認者 (Confirmer) |
| :--- | :--- | :--- | :--- | :--- |
| **毎週** | リスト | C列 | M列 (C+3列目) | N列 (C+4列目) |
| **新規** | 新規 | C列 | M列 (C+3列目) | N列 (C+4列目) |
| **特殊** | 特殊配信マスタ | F列 | L列 (F+3列目※) | M列 (F+4列目※) |
*※特殊配信枠の計算上、F列基準の相対位置で転記される*

### セクション判定ロジック
- **B列のセクション名**: 行の左端（B列）にあるキーワードを基に抽出範囲を決定する。
  - 「毎週」: リストマスタを参照
  - 「新規」: 新規マスタを参照
  - 「特殊」: 特殊配信マスタを参照
- **時間枠（A列）の継続**: B列が空欄であっても、A列に時間が記載されている限り、その行は直上のセクションを引き継ぐ。

### 確定版生成ルール
- 確定版作成時に、`INDEX(FILTER(...))` 数式が「その時点の計算結果（値とハイパーリンク）」に置き換えられ、マスタ更新の影響を受けない「静的な記録」として保存される。

## 5. 重要ファイル
- **Code.js**: スプレッドシートに実装されているGASコードのマスター。
- **【ウキ】新メルマガスケジュール(作成中).xlsx**: ローカルでのバックアップおよびデータ確認用。
- **csv_output/**: 各シートを解析用に書き出したCSVデータ。
