# メルマガ運用・開発仕様書 (SPEC.md)

## 1. プロジェクト概要
- **名称**: mail-magazine-maker
- **目的**: 社内用メルマガスケジュールの可視化と共有。
- **基本方針（最優先）**: ユーザーの指示に基づき、運用ルール、システム仕様、コードを適宜・即座にアップデートする。

## 2. 運用ルール (Excel/Spreadsheet)
### 2.1 データソース
- **新規 / リスト / 特殊配信マスタ**: メルマガの配信スケジュールデータ。
- **PR管理**: PR文章のマスターデータ。

### 2.2 PR挿入ルール（NEW）
- メルマガ内容（タイトル等）に `PR128` などの識別子が含まれている場合、**「PR管理」シートから該当するPR内容を抽出して表示**する。
- Webサイト上では、本来のタイトルに加えて、PR文章も詳細（Description）として確認できるようにする。

### 2.3 ステータスと色定義
- **赤**: 設定・確認完了（確定）
- **グレー**: 配信停止 または 未着手
- **水色**: RPA設定者（PA）の作業範囲
- **緑 / ピンク**: 週替わりの配信ターゲット（ローテーション）

## 3. システム仕様
- **データ抽出**: 複数シートのデータを統合。PR識別子がある場合は「PR管理」シートと紐付け（VLOOKUP的な処理）を行う。
- **表示**: FullCalendarを使用し、PR文章を含めた詳細情報を表示する。
